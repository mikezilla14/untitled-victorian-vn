# VNCCS Sprite Sheets & Slicing Manual

This document details the workflow for managing, slicing, and deploying high-volume grid sprite sheets generated by the Visual Novel Character Creator System (VNCCS).

---

## 📸 VNCCS Sprite Sheet Mechanics

VNCCS render cycles generate highly dense visual sheets composed of up to **12 candidate sprite positions arranged in a grid** (e.g. 3 rows by 4 columns).

### MVP vs. Backlog Strategy:
*   **Current MVP Release**: We extract pre-rendered, complete visual sprites directly from grid cells.
*   **Future Paper-Doll Backlog**: We preserve raw VNCCS files and coordinate maps so that prospective layered sprite engines can cleanly isolate individual hair, eye, limb, or dress textures down the line.

---

## 🗂️ Storage Directory Structure

To support both MVP extraction and paper-doll preservation, follow this directory pattern under `assets_source/`:

```text
assets_source/
├── vnccs_sheets/
│   └── {character}/                        # Original uncropped grid sheets (*.png)
└── sliced/
    └── {character}/
        └── {pose_family}/
            ├── raw_crops/                  # Uncleaned, bounding-box crops of individual cells
            └── final/                      # Cleaned, masked, canvas-aligned, compressed production assets
```

*Final output destination for Ren'Py deployment:*
`renpy_project/game/images/sprites/{character}/`

---

## ✂️ Grid Slicing & Extraction Workflow

### 1. Alpha-Component Extraction (Background Removal)
*   Because VNCCS outputs typically feature high-fidelity transparent channels, prefer **alpha-channel masking** rather than traditional chroma keying.
*   If a solid-colored backup background is present, use clean color-distance alpha extraction (typically targeting flat neutral gray `#808080` or flat chroma key green `#00FF00`) to isolate the character contour with anti-aliased feathered edges.

### 2. Canvas-Alignment (Preventing Sprite Jumping)
*   **The Problem**: If individual cell crops are exported using tight bounding boxes, the character will "jump" or wiggle erratically on the screen when expressions or poses are swapped in Ren'Py.
*   **The Solution**: Place every sliced character pose family on a uniform, consistent transparent canvas size (e.g. `800 × 1080 pixels`).
*   **Alignment Anchor**: Choose a static visual anchor (typically the nose tip or heels) and align it to identical pixel coordinates across every single expression variant in that pose family.

---

## 🧬 Pose Families and Expression Compatibility

Expressions and clothing assets are only modular and reusable within **compatible Pose/Head-Angle Families**:

*   Group VNCCS sheets by pose families (e.g. `standing_front`, `writing_side`, `sitting_three_quarters`).
*   Never mix cell extractions from different head angles into the same expression group. The slight perspective tilt will break the player's spatial orientation and look unprofessional.

---

## 📇 Mandatory Backlog Metadata logging

When cataloging sliced crops in a character card under `assets_source/character_cards/`, you must document the following metadata parameters:

```yaml
source_sheet: "cora_standing_family_v001.png"   # Original uncropped sheet
selected_cell: "Row 2, Column 3"               # Raw grid coordinates
pose_family: "standing_front"                  # Body classification
outfit: "maid"                                 # Costume classification
expression: "guarded"                          # Facial state
canvas_alignment_anchor: "heel_pixel_400"      # Coordinate anchor points
```
This guarantees that future developers can trace any individual sprite back to its raw generation source with absolute precision.
